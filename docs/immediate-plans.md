## Phase 2: Multi-Source Commands + Dev Install

### 2.1: Multi-Source Command Loading

**Problem:** CLI only loads from `mg_core`. Need to also check `mg_project`.

**Solution:** Use `importlib` to load `mg_project/commands/__init__.py` as a module, same as mg_core. Try sources in precedence order, track which sources were checked.

**Source availability:**
- `mg_core` - Always available (installed package)
- `mg_project` - Only if in mg-managed repo (requires mg_root)
- `mg_user` - Deferred (requires user identity)

**Error reporting:** On "command not found", show which sources were checked and which couldn't be checked (with reason):
```
Unknown command: foo

Checked:
  ✓ mg_core/commands/

Could not check:
  ✗ mg_project/commands/ - Not in mg-managed repo. Run 'mg start' or set MG_ROOT.
```

**Implementation:**
- [ ] Add `load_commands_module(init_file: Path) -> ModuleType | None` to loader.py
  - Returns None if file doesn't exist (no exception)
  - Uses `importlib.util.spec_from_file_location` like `load_command`
- [ ] Add `CommandSource` dataclass to track checked/unchecked sources
- [ ] Update CLI to try sources in precedence order
  - Catch errors from unavailable sources, record reason
  - Use `Paths.project.commands` to get path (not hardcoded)
  - Try mg_project first, mg_core second
- [ ] On "not found", print helpful message showing all sources
- [ ] Ensure scaffold creates `mg_project/commands/__init__.py`

### 2.2: Dev Install Command

Create `mg dev install` as a project command (in src/mg_project/commands/).

**Command behavior:**
- `--branch <name>` - which worktree to use (default: "main")
- Searches for `worktrees/{branch}` from mg_root
- Errors if worktree doesn't exist
- `--force` - overwrite existing installation
- Warns and exits if mg already installed (unless --force)

**Output:** Shell script at `~/.local/bin/mg`:
```bash
#!/bin/bash
# Generated by mg dev install
# Source: {{ mg_root }}/worktrees/{{ branch }}
exec uv run --project {{ mg_root }}/worktrees/{{ branch }}/mg-cli mg "$@"
```
